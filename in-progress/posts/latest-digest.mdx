## Non-widening literal type

https://mariusschulz.com/blog/const-assertions-in-literal-expressions-in-typescript

I'm wanting to make sure that you can only update a react component state or context value with a valid string value. The valid string values I keep in an object like this:

```javascript
export const EDITING = {
  NAME: 'name',
  LOGO: 'logo',
  HEADLINE: 'headline',
  SHIPPED_ON_ISO: 'shippedOnISO',
  USERNAME: 'username',
  SHIRT_COLOR: 'size',
  SHIRT_SIZE: 'color'
}

updateState({ editing: EDITING.SHIRT_SIZE })
```

The typescript error on `updateState()` is:
```shell script
TS2322: Type 'string' is not assignable to type '"name" | "color" | "logo" | "headline" | "shippedOnISO" | "username" | "size" | undefined'.
```

And after 10 minutes of looking, I found the link above and now know to use:

1. [A const assertion for literal expressions](https://mariusschulz.com/blog/const-assertions-in-literal-expressions-in-typescript#const-assertions-for-literal-expressions)
2. [A series of non-widening literal types](https://mariusschulz.com/blog/const-assertions-in-literal-expressions-in-typescript#non-widening-literal-types)
3. [Or, typescript enums](https://mariusschulz.com/blog/const-assertions-in-literal-expressions-in-typescript#using-typescript-enums)

Good to know.
