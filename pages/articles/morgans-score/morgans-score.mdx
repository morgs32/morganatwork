# Morgan's Score

A contemporary version of the [Joel test](https://www.joelonsoftware.com/2000/08/09/the-joel-test-12-steps-to-better-code/)

1. Can you start up a local development environment in one step?
1. Can you work on the frontend and backend for a feature simultaneously?
1. Can you make a build in one step?
1. Do you make daily builds?
1. Do you have a bug database? Do you fix bugs before writing new code?
1. Do you have a spec?
1. Do you have an up-to-date schedule? What's your commitment against delivery ratio?
1. Do you use the best tools money can buy?
1. Do you have ~~testers~~ test engineers?
1. Do new candidates write code during their interview?
1. Do you do hallway usability testing?

---

## 1\. Can you start up a local development environment in one step? <Anchor id="local-dev-environment" />

Joel's test step 2 is all about reducing the time and inconvenience of building from source -
or in other words reducing the time to **finish work**. I think it's even more important
to reduce the time it takes to **start work**. I'm fairly certain that it's a universal truth,
and evidence is the "Getting started" header at the top of every Readme - which more often than not
is just two lines:
```
yarn
yarn start
```
That should be all it takes to start working on a somewhat [full-featured](#full-featured-sandbox)
 sandbox'd version of your project.

## 3\. Can you make a build in one step?

Questions for an interviewer:
- How long does a deploy take?
- Do you run tests on every push to source control?


## 4\. Do you have a bug database? Do you fix bugs before writing new code?

Questions for an interviewer:
- How many unack'd bugs in your error logging service?
- What ways to you have of measuring code health? (ex. Code Climate, linters, Coveralls)
- Draw an emoji face that represents the state of your code base

## 5\. Do you have high fidelity mockups for every feature?

Questions for an interviewer:
- What kind of specs do you have for every feature ticket?
- Do you use clickable prototypes? (ex. Invision, Zeplin, Figma, Framer)
- Do features specs roll up into a complete prototype for the whole product?
- Have there been any usability tests before engineering starts work on a feature?
